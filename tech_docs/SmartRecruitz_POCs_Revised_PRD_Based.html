POC 1: Resume Parser & Skill Normalization Agent

Objective

Parse unstructured resume documents (PDF, DOCX, TXT), extract structured candidate profile data, and normalize skills to a standard taxonomy with proficiency levels. This combined agent handles both document extraction and skill standardization in a single flow.

Problem Statement

Candidates upload resumes in various formats with inconsistent structures. Skills are listed inconsistently ("JS", "JavaScript", "ECMAScript" are the same). Manual data entry takes 15-20 minutes per candidate and is error-prone. SmartRecruitz needs to create "structured, verified candidate dashboards" with "Normalized skill indicators for comparability across candidates" (PRD 6.2.1, 6.2.2).

Expected Outcome

  • Extract personal info, education, experience, skills from any resume format
  • Identify domain for each experience (FinTech, Healthcare, E-Commerce, etc.)
  • Calculate domain-wise experience (years/months per domain)
  • Extract certifications, projects, languages, and all available information
  • Normalize all skill variations to standard names (JS → JavaScript)
  • Calculate proficiency levels: Beginner/Intermediate/Advanced/Expert
  • Infer implied skills from experience descriptions
  • Create structured candidate profile ready for SmartRecruitz dashboard
  • 95% accuracy on standard resume formats
  • Processing time: under 15 seconds per resume (including skill normalization)

Chosen Approach

AI-Powered Resume Parsing + Skill Normalization with Claude 3.5 Sonnet (Vision)

Step 1: Document Extraction

  1. Upload resume file (PDF/DOCX/Image) directly to Claude AI
  2. AI reads document using vision capabilities (no separate parser needed)
  3. AI extracts all fields into structured JSON schema

Step 2: Skill Normalization (within same flow)

  1. Send raw skills + experience context to Claude AI
  2. AI maps skill variations to standard names (JS → JavaScript)
  3. AI analyzes experience descriptions to assess proficiency levels
  4. AI infers implied skills from project descriptions (e.g., "Built REST APIs" → HTTP, JSON, API Design)
  5. Output normalized skill profile with evidence-backed proficiency

Step 3: Store & Validate

  1. Validate extracted + normalized data
  2. Store complete profile in database

Note: Both extraction and normalization happen in a single AI call using Claude Vision's multimodal capabilities.

Justification vs Alternatives

AlternativeWhy Not
PDF Parser + NLPTwo-step process, parser errors propagate, maintenance overhead
Regex/Rule-based parsingFails on varied formats, needs constant maintenance
Traditional NLP (spaCy)Needs training data, lower accuracy, can't read files
Off-shelf parsers (Sovren)Expensive ($), less customizable, external dependency

Why AI (Claude Vision):

  • Reads PDF/DOCX/Images directly - no external parser needed
  • Single step: File → AI → Structured Data
  • Works with ANY resume format without templates
  • Understands context, not just keywords
  • Handles handwritten, scanned, and poorly formatted resumes

POC 2: Duplicate Identification Agent

Objective

Perform deep duplicate analysis on candidates in candidates_staging table (from POC 1) before moving them to the main database. Prevents duplicate profiles in SmartRecruitz. PRD Section 11.2: "System must prevent creation of duplicate profiles."

Problem Statement

Candidates re-apply with different emails, updated resumes, or name variations. Simple email matching misses 60% of duplicates. Duplicates cause wasted screening effort and data quality issues.

Expected Outcome

  • Detect 95%+ of duplicate profiles
  • Handle name variations (Bob = Robert)
  • Match on experience fingerprint when email differs
  • Provide evidence for why profiles match
  • Unique candidates → Move to candidates_main table
  • Duplicates → Flag for merge/reject/manual review

Chosen Approach

Two-Step Process: Database Query + AI Verification

StepActionDone By
Step 1 Query database to find potential matching candidates using:
  • Exact match on email
  • Exact match on phone (normalized)
  • Fuzzy match on name (Levenshtein distance)
  • Similar experience (company + dates)
 Database Query
(SQL/Elasticsearch)
Step 2 Send new candidate + potential matches to AI for:
  • Semantic name comparison (Bob = Robert)
  • Experience timeline validation
  • Final duplicate confidence score
 AI (Claude)

Key Point: AI receives BOTH new candidate data AND existing potential matches from database. AI does NOT have direct database access.

POC 3: Verification Agent (Identity + Experience)

Objective

Verify candidate identity by validating uploaded documents (ID proof, PAN card, Aadhaar) against claimed profile data. PRD Section 6.2.1: "Verified candidate identity onboarding."

Problem Statement

Candidates may provide false identity information. Manual document verification is time-consuming and inconsistent. Need automated verification to establish trust foundation.

Chosen Approach

Claude Vision API for Document Verification

Using Claude 3.5 Sonnet (Vision) via Anthropic API to perform OCR extraction, detect tampering/fraud indicators, and validate document authenticity through advanced image analysis.

  1. Candidate uploads ID document image (Aadhaar, PAN, Passport)
  2. Send document image to Claude Vision API
  3. Claude extracts text data using multimodal OCR capabilities
  4. Claude analyzes document for tampering indicators (fonts, edges, watermarks)
  5. Compare extracted data against claimed data from registration form
  6. Return verification result with confidence score

POC 4: Interview Question Generation Agent

Objective

Generate personalized L1 interview questions based on candidate's specific experience, skills, and knowledge areas. Ensure questions are unique per candidate and not repeated across different interviews to maintain assessment integrity.

Problem Statement

Current L1 interviews use generic question banks that don't target candidate-specific experience. Same questions are asked to all candidates, leading to leaked questions and memorized answers. Interviewers waste time asking about irrelevant areas. PRD requires "Standardized yet personalized evaluation" - questions should be consistent in quality but tailored to individual candidates.

Expected Outcome

  • Generate 10-15 personalized questions per candidate based on their profile
  • Questions target areas where candidate claims expertise (to verify claims)
  • Each candidate gets unique questions - not repeated from other candidates
  • Questions maintain consistent difficulty level per role requirement
  • Reduce question leakage risk by 90%+
  • Interview preparation time reduced from 15 mins to 2 mins
  • L1 Interview: Questions based on candidate's target role (generic template)
  • Additional Interview: Questions based on actual JD from Hiring Manager

Chosen Approach

AI-Powered Personalized Question Generation with Deduplication (Two-Stage)

Using Claude 3.5 Sonnet (Anthropic API) or GPT-4.0 (OpenAI API) for intelligent question generation. AI Model: To Be Finalized

Stage 1: L1 Interview (Before Talent Pool)

  1. Retrieve candidate's skill profile and experience (from POC 1)
  2. Get candidate's Target Role (self-declared during registration)
  3. Load Generic Role Template from master database (e.g., "Senior Python Developer" requirements)
  4. Send profile + generic requirements to AI model
  5. AI generates questions to validate claimed skills
  6. Check against history database to avoid repetition
  7. Output: L1 Screening Questions

Stage 2: Additional Interview (After POC 6 Selection - If Required)

  1. Hiring Manager selects candidate from Talent Pool (POC 6)
  2. HM requests additional test for specific job
  3. Load Actual JD requirements from Hiring Manager's job posting
  4. Send profile + job-specific requirements to AI model
  5. AI generates job-specific questions
  6. Output: Job-Specific Questions

Note: Final selection between Claude 3.5 Sonnet and GPT-4.0 to be determined based on POC evaluation (quality, cost, integration factors).

Justification vs Alternatives

AlternativeWhy Not
Static Question BankSame questions for all = leaked questions, memorized answers
Random Selection from PoolMay ask irrelevant questions, wastes interview time
Manual Question WritingTime-consuming (15+ mins/candidate), inconsistent quality
Template-based GenerationLimited variation, still predictable patterns

Why AI (Claude/GPT-4): AI can analyze candidate's unique experience (e.g., "built microservices at scale") and generate specific questions ("Explain how you handled service discovery in your microservices architecture"). Infinite variation, consistent quality, targets actual claimed skills.

POC 5: Interview Validation & Readiness Scoring Agent

Objective

Analyze L1 (first-level screening) interview recordings/transcripts to provide standardized, objective interview scores. PRD Section 6.2.3: "Standardized evaluation signals independent of individual recruiters."

Problem Statement

L1 interviews are inconsistent across interviewers. No standardized scoring. Manual review takes 30-45 minutes. PRD requires "Consistent readiness assessment" through structured validation.

Chosen Approach

Claude AI-Powered Interview Transcript Analysis

Using Claude 3.5 Sonnet via Anthropic API to analyze interview transcripts, evaluate technical accuracy, assess communication quality, and provide standardized scoring independent of interviewer bias.

  1. Receive interview transcript (from video call transcription or manual entry)
  2. Retrieve expected answers from POC 4 question generation output
  3. Send transcript + expected answers to Claude AI
  4. Claude evaluates each answer against expected answer points
  5. Claude assesses communication clarity and technical accuracy
  6. Generate standardized scores with detailed breakdown

POC 6: Job Matching & Candidate Search Agent

Objective

Match verified candidates from talent pools to enterprise job requirements with explainable scoring. PRD: "Talent Pooling & Discovery - Controlled access to enterprise consumers of talent data."

Problem Statement

Hiring managers struggle to find the right candidates from large talent pools. Manual searching is time-consuming and keyword-based matching misses qualified candidates with different terminology. Enterprises need AI-powered search with semantic understanding and explainable match scores.

Expected Outcome

  • Search candidates by JD upload, skills, domain, or manual criteria
  • AI-powered semantic matching (not just keyword match)
  • Match percentage with explainable breakdown
  • Ranked candidate list with strengths/gaps for each
  • Support for additional interview requests for selected candidates
  • Search response time under 5 seconds for pools up to 10,000 candidates

Chosen Approach

Two-Step Process: Database Query + AI Scoring

StepActionDone By
Step 1 Retrieve data from database:
  • Get job requirements (from job posting)
  • Get candidates from talent pool with matching primary skills
  • Filter by basic criteria (experience years, location)
 Database Query
(SQL)
Step 2 Send job + candidates to AI for:
  • Semantic skill matching (not just keyword)
  • Experience relevance scoring
  • Transferable skills identification
  • Explainable match reasoning
 AI (Claude)

Key Point: Database does initial filtering. AI receives filtered candidates + job requirements for intelligent matching.

Justification vs Alternatives

AlternativeWhy Not
Keyword-only SearchMisses candidates with equivalent skills using different terms (JS vs JavaScript)
Manual ScreeningTime-consuming for large pools, inconsistent evaluation criteria
Rule-based MatchingCannot handle nuanced requirements, no transferable skill recognition

Why AI (Claude): Semantic understanding of skills, identifies transferable experience, provides explainable reasoning for each match score, handles domain-specific terminology.
